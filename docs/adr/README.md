# ADR

Architecture Decision Record (ADR) files store important decisions.

## Status
- `Proposed`: discussed but not accepted yet.
- `Accepted`: agreed and active.
- `Superseded`: replaced by a newer ADR.
- `Deprecated`: no longer recommended.

## Naming
- `0001-...md`, `0002-...md`, and so on.
- Keep title short and concrete.

## Minimum Sections
- Context
- Decision
- Consequences
- Validation
